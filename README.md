## О проекте

Game Library — десктопное приложение на Electron и обычном JavaScript. Ядро
собрано в чистый ESM-модуль, поверх которого лежит безрамочный интерфейс.

**Поиск** Находит раздачи по запросу и фильтру по форуму, после чего можно
открыть тему, скопировать magnet или сохранить `.torrent`.

**Загрузки** Запускает загрузки через WebTorrent, ставит их на паузу и
возобновляет, а прогресс сохраняется между перезапусками приложения. Метаданные
`.torrent` хранятся локально, чтобы незавершённые загрузки можно было вернуть.

**Библиотека** Хранит сохранённые magnet-ссылки и скачанные `.torrent`-файлы
в локальном разделе с быстрым доступом к записи и её источнику.

**Вход** Первый запуск открывает экран авторизации. Учётные данные хранятся
локально через `safeStorage`, а сессии привязаны к активному зеркалу.

**Переключение зеркал** Если текущее зеркало падает, клиент автоматически
переходит на следующее доступное и повторяет запрос после нового входа.

**Интерфейс** Терминальный стиль с кастомной верхней панелью, поиском в
первую очередь, вкладками загрузок и библиотеки, а также разделом настроек.

### Стек

```
src/
├── client.js     Клиент GameLibrary: вход, поиск, тема, magnet, .torrent,
│                 переключение зеркал и работа с сессией
├── parser.js     Помощники для разбора HTML-страниц поиска и темы
├── mirrors.js    Список зеркал и проверка доступности
├── cookieJar.js  Минимальное хранилище cookie между запросами и редиректами
├── encoding.js   Работа с формами и текстом в windows-1251
├── errors.js     Классы ошибок для авторизации, разбора и зеркал
└── index.js      Публичный API

electron/
├── main.cjs      Главный процесс: окно, IPC, хранилище, загрузки и WebTorrent
├── preload.cjs   Безопасный мост, доступный рендереру
├── store.cjs     Настройки, статистика, библиотека и зашифрованные креды
└── torrents.cjs  Обёртка WebTorrent с паузой, возобновлением и удалением

renderer/
├── index.html    Каркас приложения: панель, вход, сайдбар, вкладки и настройки
├── app.js        Контроллер интерфейса и логика рендера
└── styles.css    Визуальная система и раскладка

bin/cli.js        Командная строка
scripts/launch.cjs Вспомогательный запуск Electron
test/             Тесты парсера и фикстуры
```

Безопасность: `contextIsolation: true`, `nodeIntegration: false` и строгий CSP.
Весь доступ к Node идёт через preload-мост и IPC-обработчики в главном
процессе.

## Установка

```bash
npm install
```

Ядро использует [`cheerio`](https://cheerio.js.org/) для разбора HTML и
[`iconv-lite`](https://github.com/ashtuchkin/iconv-lite) для декодирования
`windows-1251`. Десктопное приложение использует Electron из
`devDependencies` или системный пакет.

### Запуск локально

```bash
npm start
```

Лаунчер в `scripts/launch.cjs` сначала ищет Electron в `node_modules`, а затем
переключается на системный бинарь.

Если Electron не скачался при установке, поставьте его вручную через
системный менеджер пакетов.

### Тесты

```bash
npm test
```

### Сборка

```bash
npm run build
```

Команда использует `electron-builder` и готовит релизные файлы в `release/`:

- Windows installer: `Game Library-<version>-x64.exe`
- Windows portable: `Game Library-<version>-portable-x64.exe`
- Linux AppImage: `Game Library-<version>-x64.AppImage`

Можно собрать платформы отдельно:

```bash
npm run build:win    # installer .exe + portable .exe
npm run build:linux  # AppImage
```

Для быстрой проверки упаковки без создания installer/AppImage используйте:

```bash
npm run pack
```

## Как это работает

1. `login()` проверяет зеркала в порядке приоритета и проходит авторизацию на
   первом доступном.
2. Запросы поиска, темы, magnet и `.torrent` идут через активное зеркало. Если
   оно падает, клиент переключается на следующее и повторяет попытку.
3. Сессионные cookie хранятся локально и могут быть сохранены и восстановлены
   для приложения Electron.
4. Состояние загрузок хранится локально, а приостановленные загрузки можно
   продолжить после перезапуска приложения.
5. В публичном API предпочтительное имя экспортируется как `GameLibrary`, а
   `RutrackerClient` остаётся для совместимости.

### Пример

```js
import { GameLibrary } from './src/index.js';

const client = new GameLibrary({
  autoSwitch: true,
  timeout: 15000,
  logger: (msg) => console.log(msg),
});

await client.login();

const results = await client.search('ubuntu 24.04');
const topic = await client.getTopic(results[0].topicId);
const magnet = await client.getMagnet(results[0].topicId);
const torrent = await client.downloadTorrent(results[0].topicId);
```

## CLI

```bash
node bin/cli.js mirrors                       # проверить доступность зеркал
node bin/cli.js search "kingdom come" --forum 0
node bin/cli.js topic 6543210                 # детали темы + magnet
node bin/cli.js magnet 6543210                # только magnet-ссылка
node bin/cli.js download 6543210 film.torrent # сохранить .torrent на диск
```

CLI читает учётные данные из `RUTRACKER_USERNAME` и `RUTRACKER_PASSWORD`,
либо из флагов `--user` и `--pass`.
